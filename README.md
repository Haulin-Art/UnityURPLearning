在2025年12月开始的Unity URP学习项目
目前已经包括：
1. 无限草，已经无限草的流体场交互
2. NS 角色流体场
3. 终末地史尔特尔渲染模拟，其下包括
- Per Object Shadow Renderer Feature，具体构架是Renderer Feature执行渲染灯光空间深度图，然后绘制一张屏幕空间的PCSS阴影，再用Compute Shader降噪
- Edge Detect Renderer Feature，有两个，一个使用后处理边缘检测，一个是实体化描边，设置了专门的描边数据Class，可以控制每个的粗细颜色等等，还可以传递基础颜色贴图，目前用的是实体化的方法
- 角色表面流水模拟
## 对于无限草部分
- 核心方案：单 Compute Buffer 分段式多类型植被管理，为不同植物分配固定索引区间（如 A 类 0-49999、B 类 50000-99999），配套 GPU 原子计数器数组，独立管控每类植被的实例数量、写入位置与渲染范围，全流程数据计算、剔除、动画均在 Compute Shader 完成，CPU 零负担。
- 解决痛点：彻底避免多植被类型多 Buffer / 多 Draw Call 带来的内存冗余与 CPU 批次开销，实现单 / 极少 Draw Call 完成全类型植被渲染，完美适配 SRP Batcher/GPU Instancing，天然支持开放世界无限地形扩展。
- 配套能力：基于地形的植被生长方向控制系统，支持「完全垂直世界向上」与「贴合地形法线垂直坡面」双模式可调节，适配陡坡、起伏地形，从根源避免草体穿模、不自然生长的问题。
- 全链路 GPU 剔除与 LOD 优化体系，其中包含Tile剔除、渐进式 LOD 与密度降级、遮挡剔除、视锥剔除，利用实例索引「近距靠前、远距靠后」的预排序特性，直接对每类植被的索引计数做百分比分级 LOD 控制，无需额外排序与数据管理；配套随距离递增的实例概率剔除，远距渐进降低植被密度，避免硬切换跳变。
- 动态交互与动画系统：角色 NS 流体场实时驱动的草体动态交互，GPU 端计算流体场对单根草的摆动、倒伏影响，实现角色走过草地的自然、无穿帮交互效果，可叠加全局风场、局部扰动风，适配不同环境氛围。
- 全流程适配 Unity URP/SRP 管线，支持 PBR 光照、软硬阴影分级控制，可无缝拓展至灌木、树木等所有植被类型。
- 暴露植被密度、风场强度、交互范围、LOD 距离等全参数化控制，支持美术同学快速调整，符合工业化项目的协作需求。
